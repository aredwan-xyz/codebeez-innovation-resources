## How Can I Contribute?

### 🐛 Reporting Issues

Found a broken link? Know of a better resource? Open an [issue](https://github.com/yourusername/awesome-ai-innovation/issues/new/choose) and use the appropriate template.

### ✨ Adding a New Resource

The most common contribution — and we love them! Here's how:

1. **Check for duplicates** — Search the README before adding
2. **Verify the resource** — Make sure it's active and high quality
3. **Follow the format** — See [Style Guide](#style-guide) below
4. **Submit a PR** — One resource per PR makes review faster

### 🏗️ Adding a New Section

Have a whole new category in mind? Open an issue first to discuss it before building it out. This avoids duplicate effort and helps us align on structure.

### 🔧 Fixing Bugs / Typos

Found a typo or formatting issue? Open a PR directly — no issue needed for small fixes.

## Pull Request Process

1. **Fork** the repository
2. **Create a branch**: `git checkout -b add/your-resource-name`
3. **Make your changes** following the style guide
4. **Test your links** — ensure all URLs are valid and accessible
5. **Commit** with a descriptive message: `Add [Resource Name] to [Section]`
6. **Open a Pull Request** with:
   - A title following: `Add [Resource Name]` or `Fix [issue]`
   - A brief description of what you're adding and why it's valuable
   - Any relevant context (your experience with the tool, etc.)

### PR Checklist

- [ ] I've read the Contributing Guide
- [ ] I've checked for duplicates
- [ ] The resource is actively maintained
- [ ] I've followed the formatting style
- [ ] All links are valid
- [ ] Description is under 100 characters
- [ ] I've added to the correct section

## Style Guide

### Resource Format

```markdown
- **[Resource Name](https://url.com)** — One-line description of what it is/does
```

**Rules:**
- Use `**bold**` for the resource name inside the link
- Use ` — ` (em dash with spaces) as the separator
- Keep descriptions to **one sentence**, under **100 characters**
- Descriptions should say what it IS, not just what it does
- No trailing period on descriptions
- Use sentence case (not Title Case) in descriptions

### Table Format

```markdown
| Column 1 | Column 2 | Column 3 |
|----------|----------|----------|
| Value    | Value    | Value    |
```

### Section Headers

Use existing header emoji styles when adding to existing sections. New sections should follow:

```markdown
## 🆕 New Section Name

> One-line tagline for the section.

Content here...
```

## What We're Looking For

✅ **Resources that are:**
- Actively maintained (updated in the last 12 months)
- Genuinely useful to practitioners, researchers, or learners
- Free or have a meaningful free tier
- Well-documented
- Widely respected in the AI/ML community

✅ **Types of additions we prioritize:**
- Newly released tools that are gaining traction
- Research papers that have become influential
- Community resources that are hard to find
- Open-source projects with real adoption

---

## What We're NOT Looking For

❌ **We will not accept:**
- Resources with broken or redirecting links
- Unmaintained or abandoned projects (no commits in 2+ years)
- Pure marketing content or landing pages
- Resources behind full paywalls with no free access
- Personal blogs without substantial AI content
- Resources that duplicate existing entries
- Self-promotion without community validation (open an issue to discuss first)
